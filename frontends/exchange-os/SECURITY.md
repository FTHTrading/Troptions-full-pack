# Security Policy

## Reporting Vulnerabilities

- Do not open public issues for secrets, vulnerabilities, or security-sensitive bugs.
- Report security issues privately to: security@fthtrading.example (placeholder).
- Include reproduction steps, affected scope, and potential impact.

## Repository Security Rules

- Do not commit secrets.
- Do not commit private keys, seed phrases, API tokens, or credential files.
- Do not commit production `.env` files.

## Supported Branch

- `main`
