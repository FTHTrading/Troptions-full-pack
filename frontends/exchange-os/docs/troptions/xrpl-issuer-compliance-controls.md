# XRPL Issuer Compliance Controls

**Status:** Unsigned Templates Only | Live Execution Disabled  
**All AccountSet operations are unsigned template generation only. No transactions are submitted.**

---

## Account Flag Reference

### asfRequireAuth (Flag: 2)

Requires that the issuer authorize each trustline before it can hold the asset.

- **Required for:** Institutional asset issuance
- **Reversible:** Yes
- **Risk if missing:** Any account can create a trustline and receive the asset without issuer review
- **Template mode only:** Yes

### asfRequireDestTag (Flag: 1)

Requires that all payments to this account include a destination tag.

- **Required for:** Exchange/custodian accounts
- **Reversible:** Yes
- **Risk if missing:** Payments without destination tags cannot be attributed to individual customers
- **Template mode only:** Yes

### asfDisableMaster (Flag: 4)

Disables the master key pair for signing. Requires a signer list (multisig) to be configured first.

- **Reversible:** Yes (only if master key is re-enabled via regular key or signer list)
- **Risk if missing:** Single point of key compromise exposure
- **Prerequisite:** Multisig signer list must be configured before enabling
- **Template mode only:** Yes

### asfNoFreeze (Flag: 6)

⚠️ **IRREVERSIBLE** — Permanently disables the issuer's ability to freeze individual trustlines or globally freeze the asset.

- **Reversible:** **NO — PERMANENT**
- **Risk of premature activation:** Permanently surrenders freeze capability; cannot be undone
- **Requires explicit acknowledgment:** Yes
- **Template mode only:** Yes

### asfGlobalFreeze (Flag: 7)

Globally freezes all trustlines for the account's issued assets. Affects all holders.

- **Reversible:** Yes (can be unfrozen unless asfNoFreeze is set)
- **Emergency use only:** Intended for emergency regulatory freeze
- **Risk:** Halts all holder transactions for issued assets
- **Template mode only:** Yes

### asfDefaultRipple (Flag: 8)

Enables rippling on all trustlines by default. Required for DEX trading of issued assets.

- **Reversible:** Yes
- **Risk if missing:** Issued asset cannot participate in XRPL DEX or payment paths
- **Template mode only:** Yes

### asfDepositAuth (Flag: 9)

Requires pre-authorization before this account can receive XRP or tokens.

- **Reversible:** Yes
- **Use case:** Prevents unsolicited deposits; increases operational security
- **Template mode only:** Yes

### asfAccountTxnID (Flag: 5)

Enables tracking of the account's most recent transaction ID to prevent replay attacks.

- **Reversible:** Yes
- **Use case:** Transaction sequence security for institutional accounts
- **Template mode only:** Yes

### asfAllowTrustLineClawback (Flag: 16)

⚠️ **IRREVERSIBLE** — Enables the issuer to claw back tokens from holder trustlines.

- **Reversible:** **NO — PERMANENT**
- **Prerequisite:** Must be set BEFORE any trustlines are created
- **Risk:** Cannot be enabled after trustlines exist; cannot be disabled once set
- **Requires explicit acknowledgment:** Yes
- **Template mode only:** Yes

---

## Multisig Requirements

The following flags require a multisig signer list to be configured before activation:

- `asfDisableMaster` — cannot disable master key without an active signer list

---

## Template Generation Safety

All `AccountSet` templates generated by this framework:

```typescript
{
  _liveExecutionAllowed: false,
  _mustSignBeforeSubmitting: true,
  TransactionType: "AccountSet",
  Account: "[ISSUER_ADDRESS]",
  SetFlag: [FLAG_NUMBER],
  // ... other fields
}
```

Templates are **unsigned**. They must be signed offline and submitted separately with appropriate authorization. Never submit templates directly from this framework.
