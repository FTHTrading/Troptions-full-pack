# Stellar Issuer Compliance Controls

**Status:** Unsigned Templates Only | Public Network Blocked  
**All SetOptions/ChangeTrust operations are unsigned templates only. No transactions are submitted.**

---

## Issuer Flag Reference

### AUTH_REQUIRED (stellar_auth_required)

Requires the issuing account to authorize each trustline before it can receive tokens.

- **Type:** Account Control
- **Reversible:** Yes
- **Required for:** Regulated asset issuance
- **Risk if missing:** Any Stellar account can hold the issued asset without issuer review
- **Template mode only:** Yes
- **Public network:** Blocked until legal review

### AUTH_REVOCABLE (stellar_auth_revocable)

Allows the issuer to revoke a previously authorized trustline.

- **Type:** Account Control
- **Reversible:** Yes
- **Required for:** Compliance-controlled asset distribution
- **Risk if missing:** Cannot revoke access from non-compliant holders
- **Template mode only:** Yes
- **Public network:** Blocked until legal review

### AUTH_IMMUTABLE (stellar_auth_immutable)

⚠️ **IRREVERSIBLE** — Permanently prevents modification of AUTH_REQUIRED and AUTH_REVOCABLE flags.

- **Reversible:** **NO — PERMANENT**
- **Type:** Account Control
- **Risk of premature activation:** Permanently locks authorization configuration; cannot change authorization policy after
- **Requires explicit acknowledgment:** Yes
- **Template mode only:** Yes

### CLAWBACK_ENABLED (stellar_clawback_enabled)

⚠️ **IRREVERSIBLE** — Enables the issuer to claw back tokens from holder accounts.

- **Reversible:** **NO — PERMANENT**
- **Type:** Account Control
- **Prerequisite:** Must be set BEFORE any trustlines are established
- **Risk:** Cannot be enabled after trustlines exist
- **Requires explicit acknowledgment:** Yes
- **Template mode only:** Yes

### Home Domain (stellar_home_domain)

Sets the home domain for the issuer account, enabling Stellar.toml discovery.

- **Type:** Configuration
- **Reversible:** Yes
- **Required for:** TOML configuration, KYC anchor discovery, asset metadata
- **Template mode only:** Yes

### Multisig Signing (stellar_multisig_signing)

Configures multisig signers and thresholds for the issuer account.

- **Type:** Security Control
- **Reversible:** Yes (signers can be modified)
- **Required for:** Institutional custody security
- **Minimum recommendation:** 2-of-3 signers for production
- **Template mode only:** Yes

### Trustline Authorization (stellar_trustline_authorization)

Grants or revokes authorization for a specific holder trustline.

- **Type:** Account Control (per-trustline)
- **Reversible:** Yes (if AUTH_IMMUTABLE is not set)
- **Required for:** Controlled asset distribution
- **Template mode only:** Yes

### Asset Issuance (stellar_asset_issuance)

Creation of asset issuance payment from issuer to distribution account.

- **Type:** Payment
- **Reversible:** No (payments are permanent)
- **Prerequisite:** AUTH_REQUIRED, CLAWBACK_ENABLED, and legal review must be complete
- **Template mode only:** Yes
- **Live execution:** Blocked

## Template Generation Safety

All templates generated by this framework include:

```typescript
{
  _liveExecutionAllowed: false,
  _mustSignBeforeSubmitting: true,
  _publicNetworkBlocked: true,
  // operation-specific fields...
}
```

Templates are **unsigned**. They must be:
1. Reviewed by qualified legal counsel
2. Signed offline with appropriate authorization
3. Submitted only after production gate clearance

## Stellar.toml Requirements

For institutional asset issuance on Stellar, `stellar.toml` must include:

```toml
[DOCUMENTATION]
ORG_NAME = "TROPTIONS"
ORG_URL = "https://troptions.com"
ORG_LEGAL_NAME = "[Legal Entity Name]"

[[PRINCIPALS]]
name = "[Authorized Officer Name]"

[[CURRENCIES]]
code = "XAG"
issuer = "GB4FHGFUTLLMS3SC5RWRK6RYBGDIUQ5NR7IGN5TWAA3QVHULJ34JGEG4"
status = "test"
desc = "[Asset description — legal review required]"
```

*Legal review required before publishing stellar.toml to production domain.*
