# TROPTIONS RWA Evidence Requirements

## Evidence Purpose

Evidence records document the proof materials required before TROPTIONS can activate any RWA
provider adapter. All adapters start with `reference_only` status and require specific evidence
before progressing.

## Path to Production Readiness Per Adapter Type

### Securities-Law-Applicable Providers
(Ondo, Maple, Centrifuge, Securitize, BUIDL, BENJI, Figure)

**Required evidence before production_ready:**
1. Provider contract (signed, executed)
2. Legal opinion — securities law, Reg D / Reg A+ / Reg S review
3. KYC/AML procedures confirmation
4. Accredited investor verification procedure
5. API credentials (officially obtained)
6. Compliance team approval
7. Internal audit review

**Critical:** Securitize and BUIDL also require transfer agent license assessment before
claiming any transfer agent capability.

### Oracle / Proof Providers
(Chainlink)

**Required evidence before production_ready:**
1. Provider contract (Chainlink Labs or node operator agreement)
2. Legal opinion — scope of use, data licensing
3. API credentials (Chainlink subscription)
4. Compliance team approval
5. Proof-of-reserve documentation scope

### Market Reference Providers
(RWA.xyz)

**Required evidence before production_ready:**
1. Data licensing agreement
2. Legal opinion — data use and attribution requirements
3. API credentials (if applicable)
4. Compliance team approval

### Blockchain Infrastructure Providers
(Figure / Provenance)

**Required evidence before production_ready:**
1. Network participation agreement
2. Legal opinion — regulatory status of Provenance Blockchain
3. Technical credentials
4. Compliance team approval

## Evidence Gap Summary — Computed Metrics

Generated by `getRwaEvidenceGapSummary()`:

- **totalRequired**: Count of all required evidence records (excluding N/A)
- **present**: Count of evidence records with status `present`
- **missing**: Count of evidence records with status `missing`
- **criticalGaps**: List of highest-priority missing evidence items

---

## Evidence Collection Process

1. Initiate provider outreach — do NOT claim relationship until contract is signed
2. Engage legal counsel — obtain written legal opinion
3. Complete internal compliance review
4. Obtain API credentials through official provider onboarding
5. Document all evidence in the evidence registry (`getMockRwaEvidenceRecords()` in production: replace with real records)
6. Update adapter gates: `hasProviderContract`, `hasCredentials`, `hasLegalReview`, `hasComplianceApproval`
7. Compliance team approves status upgrade

---

## Important: No Fake Evidence

Evidence records in the current codebase are generated by `getMockRwaEvidenceRecords()` for
architecture purposes. Before any production deployment:

- Replace mock evidence generation with real evidence storage (database or document store)
- Each evidence record must have a document reference, timestamp, and reviewer
- Evidence records must be auditable
