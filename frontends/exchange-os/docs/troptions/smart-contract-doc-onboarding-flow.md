# Smart Contract and Document Onboarding Flow

## Goal
Convert project paperwork into a simple, auditable milestone flow that clients and brokers can understand.

## Phase 1: Intake and Index
- Collect term sheet, parties, and signatures
- Build document index (owner, version, status)
- Confirm authority for each signer

## Phase 2: Milestone Mapping
- Define milestone triggers:
  - Deposit received
  - Diligence complete
  - Title clear
  - Closing package signed
- Define evidence needed for each trigger

## Phase 3: Rule and Exception Design
- If/then release rules per milestone
- Exception path for disputes, missing docs, or deadline slips
- Amendment process and approval owner

## Phase 4: Approval Gates
- Legal review
- Compliance review
- Operational approval
- Governance approval (where required)

## Phase 5: Controlled Activation
- Enable only approved milestones
- Keep non-approved releases blocked
- Record all decisions in audit log

## Phase 6: Settlement and Reporting
- Execute approved settlement steps
- Capture release receipt and references
- Issue post-close report with obligation tracker

## One-Line Client Explanation
"We map your paperwork into milestone rules so funds only move when agreed conditions are met and approved."
