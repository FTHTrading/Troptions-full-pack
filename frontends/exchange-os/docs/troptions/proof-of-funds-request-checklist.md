# Proof of Funds Request Checklist

## Purpose

Use this checklist when sending a professional POF package to brokers, sellers, lenders, and title/escrow teams.

## 1) Cover Letter Fields (Required)

- Client/entity legal name
- Deal reference (property/hotel name + address)
- POF amount and currency
- Purpose of funds (acquisition/refi/bridge)
- Validity period and expiration date
- Verification contact details

## 2) Supporting Evidence (Required)

- Recent bank or brokerage statement (redacted where appropriate)
- Source-of-funds declaration
- Entity signatory authority document
- Counterparty release permissions (if under NDA)

### Digital-Asset Addendum (Required when crypto/XRPL assets are used)

- On-chain verification report with transaction hash list and validated ledger index
- Asset identity check showing both currency code and issuer address
- Approved issuer registry match (for example, official issuer class)
- Current holder trustline and balance snapshot (`account_lines` evidence)
- Signed wallet-control challenge and signature verification output
- Source-of-funds memo linked to the verified wallet and amount
- Compliance/legal sign-off that digital-asset PoF format is accepted for the deal

## 3) Compliance Controls

- KYC/KYB complete for requesting party
- Source-of-funds reviewed
- No contradictory amount figures across documents
- Release approved by compliance/legal owner
- If asset is issued token, issuer-address classification reviewed and approved
- Wallet-control signature evidence verified and archived

## 4) Delivery Format

- PDF package naming: POF_ClientName_DealName_YYYYMMDD.pdf
- Delivery channel: secure email or approved data room
- Delivery log entry recorded

## 5) No-Overpromise Language

Use this exact line:
"This proof-of-funds package is provided for transaction diligence and does not constitute a guarantee of financing, closing, or final approval."

## 6) Final Send Checklist

- Cover letter complete and signed
- Evidence attached and date-valid
- Contact verification included
- Compliance release noted
- Delivery confirmation captured

## 7) Classification Guardrails

- Do not classify an asset as "official issuer" unless issuer address matches approved registry.
- Do not classify PoF as complete if wallet-control signed challenge is missing.
- Do not use "funds proven for closing" wording until compliance/legal sign-off is complete.
