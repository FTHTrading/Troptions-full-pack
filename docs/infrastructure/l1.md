# TROPTIONS L1 (`l1/`)

Rust workspace on port **9944** (JSON-RPC) and **9945** (Prometheus metrics).

## Crates (10 under `l1/crates/`)

| Crate | Role |
|-------|------|
| `primitives` | Accounts, amounts, tx hashing |
| `crypto` | Ed25519 signing |
| `state` | Balances, soulbounds, settlements, **RocksDB persistence** |
| `runtime` | Transaction execution, **multisig** debits |
| `sequencer` | **Single-node ordering** (not BFT) |
| `soulbound` | Credential mint/revoke |
| `settlement` | HTLC / time locks |
| `atomic-router` | Batch atomic swaps |
| `governance` | On-chain proposals/votes |
| `rpc` | Query + submit + metrics + staged x402 registry |
| `node` | `troptions-node` binary |

Integration tests: [`l1/tests/integration/`](https://github.com/fthtrading/Troptions-full-pack/tree/main/l1/tests/integration).

## Consensus honesty

- **Today:** **Sovereign Sequencer** — one `troptions-node` orders transactions locally.
- **Not claimed:** Byzantine fault tolerance, permissionless validators, multi-sequencer finality, or live fraud-proof disputes.
- **Roadmap:** Fraud-proof design ([fraud proofs]({{ '/design/fraud-proofs.html' | relative_url }})) and BFT quorum **Q4 2026** — see [roadmap]({{ '/ROADMAP.html' | relative_url }}).

## DAO RPC (L1 source of truth)

| Method | Role |
|--------|------|
| `dao_getProposals` | List on-chain proposals |
| `dao_getVotes` | Votes for `proposal_id` |
| `treasury_getBalance` | Treasury balance by chain/asset |
| `dao_submit_proposal` | Signed proposal create (Ed25519) |
| `dao_cast_vote` | Signed vote |
| `dao_execute` | Signed execution |

Dashboard and `dao-service` **read** via these methods; SQLite in `dao-service` is **audit-only**.

## TLS path (local prod template)

nginx terminates HTTPS and proxies `/l1/` → 9944. See [`docker/nginx/`](https://github.com/fthtrading/Troptions-full-pack/tree/main/docker/nginx).

## Persistence

Set `L1_DATA_DIR` to enable RocksDB snapshots (state, soulbounds, settlements, treasury, x402 agent registry metadata).

```powershell
cd l1
cargo test --workspace
cargo run -p node
```

## Signed submit

`submit_transaction` requires Ed25519 signatures; invalid signatures reject at runtime. Verified by `signed_submit` integration test.
